Christopher Armenio wrote:
> Sven,
> Thanks for your response!
> 
>>From my experience using the can4linux drivers, they recommended using
> the ATCANMINI_PELICAN driver and I've seen a lot of people using the
> pipcan driver for pilcan compatible cards. What would be the proper
> procedure (assuming I can just use the pipcan driver) to create the
> can device? Is it as simple as `modprobe pipcan`?
> 
> I've tried that and I a kernel message saying "pipcan invalid base
> address". Is there a way to specify the base address anywhere?

What is needed is a SJA1000 driver module supporting the ISA bus (legacy
ioport or memory mapped devices). The pilcan belongs to that category,
but the hardware provides resources to determine the ioport and irq by
software and does a hw-specific reset. For testing purposes, you may
want to try the old "io" driver still available in the SVN repository:

  insmod sja1000-io.ko speed=125 irq=7 base=0x314

Wolfgang.
