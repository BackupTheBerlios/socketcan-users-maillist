Christopher Armenio <arsinio at gmail.com> wrote:

> When I insert the sja1000 module, I get no positive feedback that it
> actually finds my board. I'm assuming I need to set a base address
> somewhere, if so, how? Also, am I correct in assuming that once the SJA1000
> driver is loaded properly, the can device is created automatically?

AFAIK there is no such thing as a generic driver for sja1000 based
devices.

As far as legacy devices are concerned I have already seen three
different kinds of address-schemes for sja1000 register access:
Memory-mapped, io-mapped and indexed io-mapped.

In addition to this different interfaces tend to be different as far as the
device reset is performed.

Thus what you need to do is to write a small driver which wraps around the
functions already provided by sja1000.ko

pipcan.c might be the example code you are looking for.
