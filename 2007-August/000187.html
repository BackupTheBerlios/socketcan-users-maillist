Hi all,

I use socketcan with Peak System drivers in a project of mine. I
needed an easy way to keep my development machines synchronized, so I
ended up making Debian packages of the Peak driver module source and
the socketcan modules + the header files. If someone else wants to use
these, the sources.list snippet is

deb http://www.osmas.info/~tmx/debian ./
deb-src http://www.osmas.info/~tmx/debian ./

As I do not want to run a patched bleeding edge kernel (anything after
2.6.18 is completely borken on my laptop), the socketcan-headers
package installs a patched version of can.h with the protocol number
definitions included. These were moved into the kernel patch in latest
versions of socketcan. I'm not exactly sure if this is the way it
should be done, but it seems to work for me.

All the best,

Teemu
