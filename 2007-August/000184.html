I had the same problem and my solution was to add some aliases to
/etc/modules.d. I have Gentoo Linux and don't know, how it works in your
distribution, but I created a file called /etc/modules.d/llcf with the
following content:

-----
alias net-pf-29 can
alias can-proto-1 can-raw
alias can-proto-2 can-bcm
alias vcan0 vcan
alias vcan1 vcan
alias vcan2 vcan
alias vcan3 vcan
-----

Then I needed to run "update-modules" and it rebuild my /etc/modprobe.conf
with the aliases. After this I had no problems with the tools in
trunk/can-utils and trunk/test.

I hope I could help you with this.
