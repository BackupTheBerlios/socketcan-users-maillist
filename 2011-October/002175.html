IMHO linux-can at vger.kernel.org is perfect.

It matches exactly the other mailing lists naming scheme, like linux-scsi ...

SocketCAN is a project name which has been created by Jan Kiska to distinguish
this CAN implementation from other (chardev) CAN concepts.

Since it is inside mainline kernel and the 'the' Linux CAN subsystem, i was
thinking about getting rid of this project name. This is now a good chance to
name it like other subsystems on vger.kernel.org
