I fully back Marc's concerns. The MCP 251x is not a good solution if you
want to run at 1MB/s or at high message rates. The SPI interface in the
kernel is still not optimal but anyway, a memory mapped approach is
always faster. Unfortunately, the SJA100 is somewhat outdated, e.g. 5V
only. Maybe the C_CAN derivatives are more appropriate for embedded
designs nowadays.
