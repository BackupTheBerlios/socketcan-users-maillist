Hi Wolfgang,

thanks for the description as i had just tapped into the same question 
last week.

Here is how it works for me (with my OSELAS Toolchain):

1. Add the PATH where we can find the cross compiler binaries ( like 
"powerpc-603e-linux-gnu-gcc" and "powerpc-603e-linux-gnu-ld" )

export PATH=/opt/OSELAS.Toolchain-1.1.0/powerpc-603e-linux-gnu/gcc-4.1.2-glibc-2.5-kernel-2.6.18/bin:$PATH

2. Go into the appropriate path (like ~/socketcan/trunk/kernel/2.6 )

3. Say "make ARCH=powerpc KERNELDIR=/home/olli/cg3.2/build-target/linux-2.6.23.1 CROSS_COMPILE=powerpc-603e-linux-gnu-"

In point 3. also 'ARCH=ppc' is valid and this commandline can be easily extended with 'CONFIG_CAN_MSCAN=m' and friends ... 

@Fabrizio: Remark that my cross compiler has a name prefix "powerpc-603e-linux-gnu-" which might be different in your case.

Thanks for the hint.
