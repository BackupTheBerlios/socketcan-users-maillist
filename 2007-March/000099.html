On Thu, Mar 15, 2007 at 12:44:13AM +0100, Oliver Hartkopp wrote:
> Since the socketcan-core mailing list had been created, the 
> socketcan-users ML is 'very calm'. The highest activity has been on the 
> socketcan-commit ML. 

Well, that's a usual pattern ;)

> User relevant topics e.g. "Magics of message filtering" have been
> posted on the socketcan-core list [i would assume this topic in
> socketcan-users]. 

If a community has developed enough to have something like 10...15 mails
per day, list splitup definitely makes sense. But at the moment we are
in a state where the users and developers are about the same people,
maybe with a few exceptions.

> So for me there could be two ways to solve this problem:
> 
> 1) Create different mailing lists and indicate clearly which problem
> class is handled inside which ML, so that every user can post and
> discuss according his special field of interest.
> 
> 2) Use exact ONE mailing list to inform/discuss on the socketcan
> project and it's technical approaches in all regarded ISO-layers. This
> ML would also catch the SVN commit mailings (on which you may also
> start a technical discussion).
> 
> Indeed regarding the traffic & requests on the socketcan project i
> would tend to (2) - using exact one mailing list (e.g. the mostly
> populated socketcan-users ML) to get in touch with *all* relevant
> project information.
> 
> So please (users & mailing lists owners) vote for your favourite
> layout by replying "1" or "2" or contribute new ideas ...

I'd vote for 2) now and 1) later, when we have more critical mass.
