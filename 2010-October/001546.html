Yes, lincan is probably much worse than socketcan in many aspects but my
colleague use it with MPC5200 as it has slightly less overhead and it
helps him in his particular application. Lincan was recently moved to a
new repository. You can find the latest version with MPC5200 at
http://ortcan.git.sourceforge.net/git/gitweb.cgi?p=ortcan/lincan;a=summary

There is also a simple library
http://ortcan.git.sourceforge.net/git/gitweb.cgi?p=ortcan/ortcan-vca;a=summary,
which offers a single API for different CAN drivers, socketcan and
lincan included.
