It should be possible to derive the system clock frequency from the IPB 
clock frequency as IPB is either 2 or 4 times the system clock (if this 
information is not available somewhere else). For the IPB clock there's 
the function mpc52xx_find_ipb_freq(). Because the system clock can 
either be 33 MHz or 27 MHz a test of the modulo of 33MHz should tell it.
I just say, because in the link you gave, the frequency is a user 
provided Kernel configuration option, which I find a bit inflexible. We 
have e.g. different board revisions running on different IPB 
frequencies, so when we used Xenomai at that time, we had to use two 
different kernels.
