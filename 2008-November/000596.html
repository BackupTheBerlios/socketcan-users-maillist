Alan Walker wrote:
> We are currently using the CAN kernel driver from ESD for their PCI 
> CAN Card.  If someone could point me to documentation on how to get 
> this kernel driver to work with the ESD CAN card it would be greatly 
> appreciated. 

Hi Alan,

using 'an ESD PCI CAN card' is really underspecified ;-)

There are several passive cards (== access a CAN controller directly via 
PCI) or active cards that have a local 'intelligent CPU' that do the CAN 
handling and usually communicate to the host CPU via DPRAM.

In the case of a passive card you may have luck when the PCI Adapter is 
similar to the adapters in the SocketCAN SVN and you can probably kick 
it to work.

This is e.g. a passive card from ESD:
http://www.esd-electronics.com/esd2004/german/products/CAN/can-pci-266_e.htm

In the case of an active card you would need detailed information on the 
CAN-CPU / Host-CPU interface (e.g. DPRAM layout) or a current Linux 
driver source from ESD. On this base you are able to build a CAN driver 
with the (simple) network device driver model that is similarly used for 
ethernet devices.

This is an e.g. active card from ESD powered by a PowerPC405:
http://www.esd-electronics.com/esd2004/german/products/CAN/can-pci-405_e.htm

Or you just ask the ESD guys, if they are also supporting the Linux 
Mainline CAN infrastructure, like many other vendors ;-)
