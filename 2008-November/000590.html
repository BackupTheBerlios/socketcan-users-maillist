chrislpnt wrote:
> I need to set the CAN bitrate and the Peak manual and using the chardev for
> this seems a little clumsy. It would be so much more elegant to be able to
> use an ioctl call. The Peak installation manual says:
>
> "Naturally a faster „ioctl()“-interface is also defined."
>
> Great, and the file pcan.h has the ioctls control codes but I still don't
> know how to use it. Is there anybody who can shed some light on this?
>   

I don't use the ioctl() stuff and only use

echo "i 0x4114 e" > /dev/pcan32

to set the bitrates.

But you may take a look into lib/src/libpcan.c for an example of how 
ioctls are used for the pcan driver.

> On that note, any news on getting full sysfs support for setting such
> parameters with the Peak interfaces? That way, I wouldn't have to resort to
> very Peak-specific calls.
>   

It probably makes more sense to split up the monolithic PEAK driver into 
several drivers like the peak_pci already is. So that you would have a 
peak_usb, peak_pcmcia, ... also.

But this is up to the PEAK guys.
