>
> So what I have been doing is 'slcan_attach -w /dev/ttyUSB0' and sending the job to the background without entering the requested keystroke.  This method works but it is awkward and frail.

Hi Robert,

when you have a modem dial-up connection to the internet the PPP-deamon 
(pppd) does all the handling for you according hist config-file:

- grab the /dev/tty?
label:
- initialize the modem
- create a connection
- authentificate
- switch the line discipline to IP
- switch it back
- hang up
- goto 'label'

Most of this is not needed for the slcan - so slcan_attach comes with 
less intelligence.

But using a /dev/tty? and switching the line discipline need's to be 
done also.

So how would you like to improve it? Create a slcand?

Improvements are appreciated :-)
