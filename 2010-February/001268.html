On Sun, Feb 7, 2010 at 11:03 AM, christian pellegrin <chripell at gmail.com> wrote:
> Ditto for mcp251x but against mainline. Just a question I noted
> studying the error frames: it looks to me that there isn't an error
> frame saying "the situation is come back to normality" (I mean
> error-active). Is this normal or I'm just missing something?

I think that error frames are saying "hey here is the current
situation: error active or error passive".
If a node is in error passive, but the situation starts to get normal
( e.g frames are transmitted/received successfully )
then tec and rec are decreased. If tec < 128 , rec < 128 then current
node enters in error active and
now if an error occurs this node will send ACTIVE ERROR frames instead
of PASSIVE ERROR frames.
