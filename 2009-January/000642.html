Hey!

I'm playing around with socket can since two months. In the this week i make
checks for sending of messages and find following behavior.

Hardware:
- Industrial PC with AMD LX800
- PCI-to-ISA Bridge IT8888
- PC-104 Card with Adress Latch Enabled

After a while (the time when it happen is independent) sending a message i
don't get a transmission complete interrupt.

So to check, if this really happen and when it happen i make a select in my
program and and add a status query with a flag.
Flag is set to true at the end of sja1000_start_xmit. While flag is true the
program is not allowed to send a new message.
When enter in transmission control interrupt flag is set to false.

So my program will not send messages if this error happen - which happens in
this case. Receiving is no problem.

I tried this on 3 different interrupts -> same situation

In my log after this error occurs i get following entries in registers:

REG_CMR=0x00
REG_SR=0x1D
REG_IR=0x01
REG_IER=0xFF

So all looks fine, but why the flag is not set back to false? What can
happen that in REG_IR the transmission interrupt bit is not set but the
transmission is already completed for a while?
Have someone same behaviors or is this normal?

I will check this once more in another industrial PC.
