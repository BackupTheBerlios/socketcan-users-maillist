Hello all,

i wanted to inform you about a minor change in the _internal_ API of the 
CAN core interacting with the CAN protocols like can-raw, can-bcm and 
can-isotp.

This change is NOT relevant for

- userspace applications
- CAN network driver programmers

Only if you write a CAN protocol inside the Kernel* there will be a 
minor change in the upcoming 2.6.29:

http://git.kernel.org/?p=linux/kernel/git/davem/net-2.6.git;a=commitdiff;h=1fa17d4ba43d7e5aab5e90777b07da06524f6748

Or see in the SocketCAN SVN:
http://svn.berlios.de/wsvn/socketcan/?op=comp&compare[]=%2F at 893&compare[]=%2F at 895

Best regards,
Oliver
