Dear List,

I'm new here and trying to study if we should move to the SocketCAN API.

We have a design with three MCP2515 parts on board and full Linux. I have used the old can4linux code as the base for my 
driver and now stumbled into the SocketCAN project. If I have understood right, this should become the standard Linux 
CAN API and, to stay current, we too should start to use it.

One concern though: We are committing safety audit on one of our projects. In that process, it has been made clear that 
generated messages must be sent in the order of priority. High priority messages must not be blocked by lower priority 
messages in software or hardware queues.

I have tried to grasp how this is in SocketCAN. Right now i'm in the belief that SocketCAN sends the messages in the 
order they are introduced to the API. Am I right in this? Is this the way it is going to be? How have others solved this 
problem?
