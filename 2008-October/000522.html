Playing with different versions is a pain on powerpc, yes.

> > So the former is essential and I have to change some startup scripts.
> > Would have maybe been good to provide a compatible canconfig program for
> > backwards compatibility ...
> 
> canconfig was never part of the Socket-CAN distribution (SVN repository).

canconfig supports the new method for kernels that support it. Note that
nobody out here has the time to care about *every* old revision one
might want to work with - we support the latest and greatest, and even
that is a bunch of work to keep up to date. Once SocketCAN went into
mainline, there is a well defined set of versions to be compatible with.
But until then it is not doable: remember that, at least in theory, you
can use *any* svn revision of SocketCAN and backport it to *any* random
version of the kernel. A maintenance nightmare.

Nevertheless, if you have patches for canconfig, please send them.

> > Is setting of the baudrate via user space API still supported ? 
> > When upgrading socket-can, do I have to recompile all user space
> > programs as well ?
> 
> The IOCTLs have been removed as there is little chance to get them
> included in the mainline kernel. Please use the SYSFS files instead.

That's what canconfig from canutils-v3 supports:
http://www.pengutronix.de/software/socket-can/download/canutils/v3.0/
