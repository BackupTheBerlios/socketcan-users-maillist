Hi Daniel,

the SocketCAN ABI is stable for about 2 years now, so the userspace 
programs you originally downloaded from the Berlios SVN in rev 679 
should still work. And the latest tools should also run on SocketCAN 
rev679. Upgrading the SocketCAN userspace tools makes sense due to 
several improvements and fixes. E.g. candump supports a much better 
per-netdevice filter handling.

The major changes in the last time are caused by the nice CAN netdevice 
library from Wolfgang that allows the configuration via sysfs (ioctl's 
for this kind of configuration do not have any chance for mainline 
Kernel anymore).

I would *recommend* to use the latest SocketCAN trunk source which 
compiles excellent with 2.6.24 without any backporting. Before copying 
the files directly into your 2.6.24 kernel tree, you should try the 
'mkpatch' or at least 'strip-src' scripts (after modifying FILES-2.6 for 
your needs).

e.g. ./mkpatch 2.6.24 < FILES-2.6-daniel > daniels.patch

I hope and assume the latest mscan is also compiling for 2.6.24, but i'm 
not able to test it here.

Anyway: The tool 'canconfig' was never part of the SocketCAN SVN.
