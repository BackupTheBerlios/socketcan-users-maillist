I just built the kernel for my TQM5200 board from DENX Linux 2.4.24 (GIT
tree linux-2.6-denx, branch DENX-v2.6.24-stable). Here is what I get:

Boot log:

  mpc52xx-mscan initializing
  mpc52xx-mscan mpc52xx-mscan.0: probe for a port 0xC506A900 done
  mpc52xx-mscan mpc52xx-mscan.1: probe for a port 0xC506C980 done

Test without cable connected or without using canconfig:

  bash-3.00# ifconfig can0 up
  bash-3.00# ifconfig can1 up

  bash-3.00# ./cansend can0 123#deadbeef

  bash-3.00# ./candump -e 0xffff any
  can0  123  [4] DE AD BE EF
  can0  20000040  [8] 40 00 E9 68 0A 00 1E D3   ERRORFRAME

  The controller reports "error passive" state".

  bash-3.00# grep can /proc/interrupts
  145:          1  MPC52xx Peripherals Edge      can0
  146:          0  MPC52xx Peripherals Edge      can1

  "ifconfig can0" and "ifconfig can1" do not show any RX or TX bytes.

With cable connected:

  bash-3.00# ./canconfig can0 -b 125000
  bash-3.00# ./canconfig can1 -b 125000

  bash-3.00# ifconfig can0 up
  bash-3.00# ifconfig can1 up

  bash-3.00# ./cansend can0 123#deadbeef

  bash-3.00# ./candump -e 0xffff any
  can0  123  [4] DE AD BE EF
  can1  123  [4] DE AD BE EF

  bash-3.00# grep can /proc/interrupts
  145:          1  MPC52xx Peripherals Edge      can0
  146:          1  MPC52xx Peripherals Edge      can1

  bash-3.00# ifconfig can0 | grep bytes
            RX bytes:0 (0.0 b)  TX bytes:4 (4.0 b)
  bash-3.00# ifconfig can1 | grep bytes
            RX bytes:4 (4.0 b)  TX bytes:0 (0.0 b)

As I already mentioned, you definitely need to set the bit-rate before
starting or using the device. I do it with canconfig but it could be
done in your application as well, using the appropriate ioctl request.

You should really use the MSCAN driver from the SVN branch
"cannetdriver26", which does not have these limitations.

Wolfgang.
