The connection orientation makes it a bit more difficult, as you have to 
implement all this bind/listen/accept stuff on the server side to have a 
similar behaviour as you know it from TCP - and this forces you to have 
two socket file descriptors dealing with one connection (one for the 
listen handling and one to really handle the data stream). On the client 
side it's pretty easy as you just need to use 'connect'.
