> And probably all problems are solved here, aren't they?

As I have showed you, there are no serious problems with that code. It's
just cumbersome to set the bitrate before you start the device.

The SVN branch "cannetdriver26" has been created to develop a common CAN
device interface for all CAN drivers. In the SVN trunk, and that's what
you have in your branch of linux-2.6-denx, there is the old stuff.

> Do I really need the cannetdriver26. And in order to get it, Do I just
> have to copy the files and substitute them on my dir?

As I said, you don't need it to get MSCAN working. But it might be wise
to switch to "cannetdriver26" to benefit from the new enhanced features
like configuring the device via SYSFS. In that case you have two options:

 - Switch to 2.6.25 of DENX linux-2.6-denx

 - Use 2.6.24 from kernel.org and the "cannetdriver26" branch from the
   SVN repository and build the modules externally.

 - Use the "cannetdriver26" branch from the SVN repository and build the
   modules externally. For that, you need to remove some related files
   from the linux-2.6-denx tree. I could provide a script if you want to
   follow that road. This is a hack, of course, which I do not really
   suggest.

Wolfgang.
