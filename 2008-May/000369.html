Hi Neal,

indeed there should be no change for Kernel 2.6.24 as you always needed 
to say

make NET=NETDEV_SUPPORT

to compile with netdev support since PEAK driver 6.0 - as NETDEV_SUPPORT 
is not the default.

The only change is that the Kernel config setting CONFIG_CAN is checked 
since 2.6.25 and when it's set, NETDEV_SUPPORT is enabled automatically.

That's exactly what's described in the Changelog on the PEAK Website:

http://www.peak-system.com/linux/index.htm
"Automaticly compiling for Socket CAN Support if the Kernel option 
"CONFIG_CAN" is set (Kernel 2.6.25 or higher)"
