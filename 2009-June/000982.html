> if I understand it right the virtual CAN Bus (vcan) is only for testing when there is no real can adapter available?
>
> Just to be sure ;-)
>   

Yes. It's a software device similar to the loopback device you might 
know from the internet protocol.

The difference is, that a full qualified CAN 'address' constists of

- the CAN identifier
- the CAN bus you can see that message

And for that reason you might need to have more vcan's in your system.

But that's no problem as you can create thousands of them with th 'ip' 
tool :-)
