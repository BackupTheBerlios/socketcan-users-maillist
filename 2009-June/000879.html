> it doesnt work still
> here is the debug msgs from the kernel.


> [  383.989052] NETDEV WATCHDOG: can0 (): transmit timed out

This looks like you don't have a CAN node the controller can talk to.

Possible problems:

- wrong bitrate
- missing CAN bus termination
- missing second CAN node with correct bitrate ;-)

When the CAN frame is not acknowledged the NETDEV WATCHDOG occurs - and this
make this ugly call trace in 2.6.29+ Kernels. The netdev watchdog is removed
in newer drivers.

Please check the three points above. The best test is to receive CAN data from
a well working CAN source first.
