Hi Wolfgang,

AFAIK the CPC-104M is no PCI design but plain memory mapped access.

There are many of these PC104 boards for SJA1000, i82527 and others that are
memory mapped or even ISA I/O port based.

Currently we only have the 'old' drivers that support these type of cards.

Is the new sja1000 pipcan driver some kind of prototype for these cases?

I wonder if it's possible to create some kind of ioport or iomem frontend for
the (new!) sja1000 driver, that can be configured via commandline to support
several ioport addresses or interrupt configurations to be used by many
standard designs that only differ in port and irq settings ??

One of these would be the CPC-104M or the PEAK PCAN-PC/104 then ...
