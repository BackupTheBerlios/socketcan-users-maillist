> It probe but only the first can at adress 0x300 but says no module found

The SJA1000 needs a memory register space of 0x20 or 0x100 bytes.

So 0x04 or 0x08 bytes are definitely too small for that.

There are hardware designs that use multiplexed register access to reduce the
needed address space to 2 or 4 bytes per chip.

As you can see on page 19 of the given documentation in

>
> http://www.eurotech.fi/products/manuals/COM-1273_manual_10.pdf
>

the register access to the controllers are handled via 3 registers!

You would need a modified version of sja1000-iomux.ko then that can cope with
the multiplexed register access given in the Eurotech spec.
