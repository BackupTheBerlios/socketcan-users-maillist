Hi all,

I'm porting an old implementation of CAL/CiA to Linux.  The device connection protocol has some pretty tight timing.  I've observed that some CAN packets are buffered for 500 ms between the socket write() call and the actual transmission of the packet.  This is enough to sabotage the connection process, and possibly take out other devices as well.  Are there any socket options to disable buffering of write packets, and send them when write() is called?
