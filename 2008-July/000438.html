while reading http://www.semiconductors.bosch.de/pdf/can2spec.pdf,
I must conclude that 'on the wire', sync_seg, prop_seg, phase_seg1 & phase_seg2
are clearly identified.
A 'stay forever' API would stick to these.
If a hardware manufacturer deviates from this API (which I understood has already
happened), there should be a way to map these 4 segments to the hardware. This, I think,
is exactly the job of the driver.
Implementing algoritms other than specified
will be subject to change, misunderstandings, ..., never complete.
