>>> +static struct ccan_dev *cdevs[NUM_CDEVS];
>>
>> why do you need a global var here?

> because as i understand there is only one platform_device structure per 
> device. as the hynix chip comes with two c_can ip's i needed some way to 

right

> store the c_can device pointers. of course i could dynamically allocate 
> a structure to store this data and assign it to the platform_device. 
> would this be the preferred way?, or is there another simple/correct way 
> to use several chips with a single platform_devices?

The concept of the drivermodel is to have exactly one (platform_)device 
per hardware. And only one driver. So in your case of 2 can devices 
(which is a special case of "any" number of can devices) you just have 
to add 2 platform_devices to you board specific file in the kernel 
(along with the irq and mem resources).

Have a look how the original drivers handles this, no usage of global 
variables. The platform_driver.probe function is calles one per platform 
drice which matches the driver.
