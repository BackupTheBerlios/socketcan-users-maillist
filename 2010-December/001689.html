Attached you find a patch against the current SVN repo for the kernel
documentation. The patch:
- contains all the remaining text translated from the German docs (BCM
  stuff)
- Rewrites can-raw.txt with the correct text
- deletes can.txt, as it is no longer needed

In can-bcm.txt RX_READ and RX_DELETE are not documented. They are not
described in the German docs and, while I can guess what they do, it is
better if someone with direct knowledge of them writes a few lines.

With this patch all the useful text I had translated has been ported.
Let me know if I can be of any help with similar stuff.

As a side note, I can happily report that the DPGS system I worked on
and that was the main cause for this documentation effort, has passed all
tests and will be deployed sometime next year.
It is based on Linux, socketcan and BCM and is used for container tracking
in harbour terminals.

Actually the patch is too big for the ml, you can find it here:
http://www.brownhat.org/docs/socketcan.html
http://www.brownhat.org/download/upatches/documentation-add-bcm-2.diff
