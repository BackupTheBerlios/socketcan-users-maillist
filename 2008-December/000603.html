Yes me ;-)

They are already supported by the slcan driver which supports a class of 
devices that implements the Lawicel protocol aka SLCAN (Serial Line CAN) 
protocol.

http://download.berlios.de/socketcan/SLCAN-API.pdf

I implemented this driver as a Linux serial line discipline (N_SLCAN 
since 2.6.25) on the base of slip.c to push the tty ASCII CAN frames 
into a slc0, slc1, ... CAN netdevices.

The slcan driver is in the SocketCAN SVN (but not default compile target).

http://svn.berlios.de/svnroot/repos/socketcan/trunk/kernel/2.6/drivers/net/can/slcan.c

If you go into

socketcan/trunk/kernel/2.6/drivers/net/can

and say

make CONFIG_CAN_SLCAN=m

it is build.

You need the tool 'slcan_attach' from socketcan/trunk/can-utils also.

But for an unknown reason it is not possible to attach, detach and 
re-attach a (USB/) serial device.

But the fist attach runs excellent for first tests. Maybe you catch the 
problem ;-)

Give it a try.
