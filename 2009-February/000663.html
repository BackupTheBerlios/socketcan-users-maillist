Hello,

I'm doing a little testing with the following setup:

- PXA270
- MCP2515
- 1Mbit/sec datarate

On one system I run: cangen can0 -v -g1
on the 2nd system I run: candump can0

If I run it with -g10, it will run all day long.  With -g1, the
receiving side soon stops, and the only way I can get it to work again
is send something (cansend can0 123#DEADBEEF), and then it will
receive again.  Is this expected?

On a related note, what is the best way to test performance.  I'm
interested in a app that:

- can be run with real-time priorities
- logs errors, etc.
- measures jitter
- can be used to determine effective bandwidth of the can bus (can we
keep the can bus fully utilized on a moderately loaded system using
kernel preemption and real-time priority apps).

If nothing exists, I'll probably start writing something.
