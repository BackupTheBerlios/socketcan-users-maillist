On 07/07/2010 07:58 AM, yigit agabey wrote:
> Hello,
> I will ask an elementary question. At the moment i am using
> CONFIG_PREEMPT_RT patch on my ADS5121 board which has mscan controller on
> it. And i am also using socketCAN. SocketCAN works perfect . But now I want
> to use SocketCAN api inside a realtime application and i need RT-SocketCAN.
> I dont know where to start exactly. I see that Xenomai and RT-SocketCAN are
> generally used together. Is it possible to use RT-SocketCAN without
> Xenomai?(just with CONFIG_PREEMPT_RT) And if "yes",  is RT-SocketCAN  ported
> to ADS5121 board.

RT-Socket-CAN is based on the "Real-Time Driver Model" [1] of the
Xenomai real-time extension and therefore it cannot be used for vanilla
or "-rt" Linux. And yes, it does support the MPC5121 [2]. Anyway,
Socket-CAN should work fine with "-rt" Linux, with soft real-time
behavior, though.

Wolfgang.

[1]
http://www.xenomai.org/documentation/xenomai-2.4/html/api/group__rtdm.html
[2]
http://www.rts.uni-hannover.de/xenomai/lxr/source/ksrc/drivers/can/mscan/Kconfig
